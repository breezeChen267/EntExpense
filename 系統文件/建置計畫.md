本計畫旨在根據《台幣費用申請管理系統建置規畫書（V3.0）》建立專案的基礎架構與核心模組，包含前後端的原始碼結構搭建、Clean Architecture 實作以及核心 Domain 物件的產生。

Proposed Changes
本階段將著重於建立專案骨架與核心驗證服務，為後續功能的開發提供基礎。

Backend 架構建立 (WFP.ExpenseAPI)
我們將建立一個 .NET 8 的空白 Solution，並切分出四層架構 (Clean Architecture)。

[NEW] 方案檔與專案
WFP.ExpenseAPI.sln
WFP.Domain (Class Library)：存放 POCO 實體與介面
WFP.Application (Class Library)：存放 DTOs、商業邏輯 (Services) 與 FluentValidation 驗證
WFP.Infrastructure (Class Library)：存放對外連線 (EF Core, Dapper) 的實作
WFP.Api (Web API)：對外暴露的 API 端點，包含 Swagger 設定
[NEW] Domain 與 Application 實作
根據規畫書的「AI 開發指南」建立基礎實作：

WFP.Domain/Enums/AplStat.cs：將申請單狀態封裝為 Enum。
WFP.Domain/Entities/WFPPAYE.cs：依據 WFPPAYE (付款主檔) 設計成 C# POCO，設定 decimal 欄位並加入基礎驗證（Required、MaxLength）。
WFP.Application/Validators/ExpenseValidationService.cs：實作核心業務邏輯（受款與請款金額平衡的 Triple Check，及支票連動檢核），回傳 Dictionary<string, string> 字典表示錯誤。
Frontend 架構建立 (WFP.ExpenseWeb)
[NEW] Angular 專案
使用 ng new WFP.Web (Angular 17+ 支援 Standalone 與 Signals)。
配置 Tailwind CSS 以滿足 UI 開發需求。
清理預設樣板檔並建立預設的專案資料夾結構 (例如 features, shared, core)。
