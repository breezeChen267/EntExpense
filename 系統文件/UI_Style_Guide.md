# 台幣費用申請管理系統 - UI/UX 視覺設計手冊

本專案採用「頂級金融儀表板 (Bloomberg Style)」設計語意，強調極高對比度、專業深色調以及高效能的輸入操作體驗。

## 1. 核心配色方案 (Color Palette)

本專案全面採用「極致深色模式 (Ultra Dark Mode)」，以營造專業且專注的財務工作環境。

| 元素 | 十六進位值 | 視覺描述 |
| :--- | :--- | :--- |
| **頁面背景** | `#020617` (Slate 950) | 基底背景顏色，消除光澤，強化內容。 |
| **卡片背景** | `#1e293b` (Slate 800) | 區塊容器背景（請款平衡、費用明細、受款資訊）。 |
| **內部輸入背景** | `#0f172a` (Slate 900) | 表格標題、輸入框容器背景，營造深層次感。 |
| **主視覺藍 (`Primary`)** | `#3b82f6` (Blue 500) | 按鈕、高亮邊框、大寫金額、核心數值。 |
| **成功綠 (`Success`)** | `#10b981` (Emerald 500) | 平衡驗證通過標記、分攤進度條完成狀態。 |
| **警告紅 (`Danger`)** | `#f43f5e` (Rose 500) | 未平衡數值高亮、刪除按鈕、警告文字。 |

## 2. 字體與排版 (Typography)

*   **字型族群**: `Microsoft JhengHei` (微軟正黑體), `Inter`, `system-ui`.
*   **字型設計**: 優先考慮財務數據的清晰度，數字部分採用 `font-mono` (等寬字型) 以利於金額對齊。

### 字級規範 (Font Sizes)
*   **受款/請款平衡數據**: `text-xl` (20px) - 使用強調色以利於 Triple Check 快速核對。
*   **受款人欄位/明細輸入**: `text-sm` (14px) - 保持緊湊感與平衡。
*   **區塊標題欄位**: `text-xs` (12px) - 採用 `slate-300` 亮灰色，加粗並增加間距。
*   **新台幣(大寫)**: `text-lg` (18px) - 居中顯示，縮短行距，強調最終審核金額。

## 3. UI 元素規格 (UI Components)

### 區塊元件 (Card Layout)
*   **間距 (Spacing)**: 採用緊湊型布局，卡片內距設定為 `p-6`，KPI 區塊設定為 `py-2 px-4` 以節省垂直空間。
*   **邊框 (Borders)**: `#334155` (Slate 700) 搭配細微的陰影 (`shadow-2xl`)，營造精密儀器感。

### 互動與按鈕
*   **操作按鍵**: 小型的 `+` 與 `×` 按鈕，統一採用 `24px (h-6)` 高度，並與輸入框等高。
*   **Hover 效果**: 所有的操作行在滑鼠懸停時會變換背景色 (`hover:bg-slate-800`)，以提供直觀的反饋。

## 4. UX 操作體驗設計 (UX Experience)

*   **純鍵盤流操作**: 
    - 支援 `TAB` 鍵在所有輸入欄位間循環。
    - **Enter 鍵連動**: 在「費用摘要」、「金額」或「應付金額」欄位按下 Enter 鍵，系統會自動新增下一行，實現「盲打」式的高效錄入。
*   **即時 Triple Check**: 使用 Angular Signals 技術，在每一筆金額跳動時，同步刷新上方三個平衡快顯卡，徹底杜絕人為算錯。
*   **中文化介面**: 所有的 Drag & Drop 與提示資訊均已中文化，降低系統學習門檻。
